# AI-Assisted Custom Application Maker Portfolio

This portfolio presents my aspirational professional identity as a custom application maker who uses AI as a collaborative tool in the design and development process.

## Professional Direction

My goal is to become a developer who builds personalized desktop applications, portfolio websites, and business-ready digital experiences. I chose this direction because I believe the most valuable software developers of the future will be those who can combine human creativity and judgment with AI-assisted workflows.

## AI Workflow

I used AI to help research professional directions, brainstorm project ideas, refine writing, plan the site structure, and improve the design. I made final decisions about the concept, style, sections, and message.

The site itself includes an AI Workflow section that explains this process transparently.

## Built With

- **Next.js 14** — React framework with App Router
- **React 18** — Component-based UI
- **Tailwind CSS** — Utility-first styling
- **AI-assisted planning** — Claude used for structure, copy, and component design
- **GitHub Pages** — Static export hosting

## Project Structure

```
/app
  layout.jsx        — root layout + Google Fonts
  page.jsx          — home page
  globals.css       — Tailwind + custom animations

/components
  Navbar.jsx        — fixed navigation with mobile menu
  Hero.jsx          — full-screen hero section
  FlyingBuilderBot.jsx — animated SVG robot helper
  About.jsx         — personal identity + tools grid
  Workflow.jsx      — 5-step AI workflow cards
  Services.jsx      — 4 service offering cards
  Projects.jsx      — featured project cards
  Timeline.jsx      — future roadmap stages
  Contact.jsx       — CTA contact section
  Footer.jsx        — site footer

/data
  projects.js       — project card data
```

## Running Locally

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Building for GitHub Pages

```bash
npm run build
```

The `out/` folder contains the static export. Upload it or configure GitHub Pages to deploy from it.
